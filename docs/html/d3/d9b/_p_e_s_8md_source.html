1 ### PES evaluations {#PES}
2 
3 CDE can call several external programs to obtain energies and forces on atoms during GDS and NEB runs. This section describes how this is managed, and how input files can be set-up to run different external codes.
4 
5 ## PES types
6 
7 - The potential energy surface (PES) type to be used during a CDE-based calculation is indicated through the variables **pestype** and **pesopttype**.
8 
9 - The **pestype** is used whenever the code requires a single-point energy evaluation.
10 
11 - The current allowed values of **pestype** are (see pes.f90):
12  - *orca*: Performs a calculation using the quantum chemical program ORCA (https://orcaforum.cec.mpg.de)
13 
14  - *dftb*: Performs a calculation using the density-functional tight-binding code DFTB+ (https://www.dftbplus.org)
15 
16  - *psi4*: Performs a calculation using the psi-4 quantum code (http://www.psicode.org)
17 
18  - *lammps*: Performs a calculation of the PES using the LAMMPS simulation package (https://lammps.sandia.gov)
19 
20  - *null*: Does not perform a PES evaluation. Simply returns V = 0 and forces(:) = 0 for all atoms.
21 
22 
23 - The **pesopttype** is used whenever the code requires a geometry optimization.
24 
25 - The allowed values of **pesopttype** are:
26  - *orca*: ORCA calculation.
27 
28  - *dftb*: DFTB+ calculation.
29 
30  - *psi4*: Psi-4 calculation.
31 
32  - *lammps*: LAMMPS calculation.
33 
34  - *uff*: Performs geometry optimization under the UFF (Universal force-field) as implemented in babel/openbabel (http://openbabel.org/wiki/Main_Page)
35 
36  - *null*: Does not perform geometry optimization.
37 
38 - To add new PES evaluation types, you need to edit the files pes.f90 and io.f90; **DO NOT DO THIS UNLESS YOU ARE SURE YOU KNOW WHAT YOU ARE DOING!**
39 
40 ## PES and geometry optimization executables
41 
42 As well as knowing which type of PES calculation you'd like, the CDE code needs to know *how* to run the external executables which are requested to calculate potential energies and forces, or to run geometry optimizations.
43 
44 The executables to be used to energy evaluations or geometry optimization are defined in the variables *pesexecutable* and *pesoptexecutable*, as follows:
45 
46  pesexecutable ~/programs/orca
47  pesoptexecutable ~/programs/orca
48 
49 In the example above, we're are indicating that the external program to be used is *orca*, and this is stored in the directory *~/programs/*.
50 
51 Internally, CDE runs FORTRAN *EXECUTE_COMMAND_LINE* calls to run external codes, using input files generated according to templates provided as *pesfile* and *pesoptfile* (see below).
52 
53 For example, based on the above, CDE will run the following command when an *ORCA* calculation is required:
54 
55  ~/programs/orca temp.in
56 
57 where *temp.in* is the name of a temporary input file which is auto-generated by CDE during NEB calculations.
58 
59 **If there is no executable called "~/programs/orca", the CDE calculation will fail!**
60 
61 To make sure everything runs smoothly, there are two useful options:
62 
63 (1) You can set up an alias for each executable, then provide the alias as *pesexecutable* and *pesoptexecutable*. For example, lets say you have *ORCA* installed in *~/programs/stuff/orca/bin/*. If you include an alias in your *.bashrc* (or similar config file if you're on a different system) which reads
64 
65  alias orca='~/programs/stuff/orca/bin/orca'
66 
67 then running the command *orca* will then execute *~/programs/stuff/orca/bin/orca*, which should indeed correspond to an executable binary.
68 
69 This means that you can then use the following in your CDE input files:
70 
71 
72  pesexecutable orca
73  pesoptexecutable orca
74 
75 without having to provide the full path to the executables.
76 
77 (2) As an alternative, you can also give the full pathname to the desired executable directly in the input file, lik this:
78 
79  pesexecutable ~/programs/stuff/orca/bin/orca
80  pesoptexecutable ~/programs/stuff/orca/bin/orca

83 ## PES template files
84 
85 - The final pieces of information which CDE requires in order to perform energy evaluations or geometry optimization is a PES *template file*.
86 
87 - The input parameters **pesfile** and **pesoptfile** are header files which are used to direct external codes which calculation to perform when they are called by CDE. These header files must have the same format as input files required by the *pesexecutable* and *pesoptexecutable*.
88 
89 - The only difference is that there must be a line with *XXX* as follows:
90 
91 
92  XXX
93 
94 - The *XXX* tells the CDE code where it should insert the coordinates of the configuration at which the energy evaluation is requested. After inserting the coordinates, CDE writes the input file then runs the relevant external executable to evaluate the energy. Once the energy evaluation (or geometry optimization) is complete, CDE reads in the results.
95 - The file pes.f90 can be consulted to see how this write/run/read process works for each external code.
96 - An example header file for a PES evaluation by ORCA using PM3 semi-empirical method might be as follows:
97 
98 
99  ! PM3
100  * xyz 0 1
101  XXX
102  *
103 
104 
105 - An example header file for PES evaluation by DFTB+ might be as follows:
106 
107 
108  Geometry = GenFormat {
109  XXX
110  }
111  Driver = {}
112  Hamiltonian = DFTB {
113  SCC = Yes
114  SCCTolerance = 1e-3
115  Mixer = Broyden{}
116  Eigensolver = RelativelyRobust {}
117  MaxAngularMomentum {
118  C = "p"
119  O = "p"
120  H = "s"
121  }
122  SlaterKosterFiles = Type2FileNames {
123  Prefix = "/Users/scott/code/cde/src/SKfiles/"
124  Separator = "-"
125  Suffix = ".skf"
126  }
127  }
128  Analysis = {
129  CalculateForces = Yes
130  }
131 
132 - Note in the above examples that the only part of the file which is written by CDE is in replacing the *XXX* with the coordinates at which the energy evaluation is being performed. All other aspects, such as the calculation type (e.g. DFT, Hartree-Fock, semi-empirical, DFTB+), basis sets, accuracy, and so on, are controlled by editing the *pesfile* and *pesoptfile* BEFORE the calculation starts.
