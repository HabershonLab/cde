1 ###Tutorial 3: AutoNEB {#autoneb}
2 
3 AutoNEB is an alternative to the typical NEB algorithm. You can find useful references to the AutoNeb method in the *References* (#References) section of the documentation.
4 
5 This example is similar to *Tutorial 2*, with the main distinction being the fact that AutoNEB is used instead of standard CINEB. In AutoNeb, after an initial NEB refinement, one then adds a series of beads into the current reaction-string, followed by further localized refinement. This approach iteratively improves the description of the MEP, and often leads to smoother reaction profiles.
6 
7 Because this tutorial is very similar to *Tutorial 1*, we refer to that page for further details about LAMMPS and ReaxFF. Here, we simply highlight the input file, which is as follows:
8 
9  # This input file demonstrates a AUTONEB refinement. We use the Quickmin optimization method here, and lammps plus Reaxff for PES calculations.
10 
11  calctype optpath
12 
13  # We start with 7 images with which to perform regular NEB
14  nimage 7
15  # We shall add a futher 23 images using AUTONEB, making a total of 30 at the end of the procedure
16  autoneb 23
17 
18  # Define initial and final images in files called start.xyz and end.xyz:
19  startfrompath .false.
20  startfile start.xyz
21  endfile end.xyz
22 
23  # Typical path optimization parameters
24  pathoptmethod cineb
25  nebmethod quickmin
26  nebiter 500
27  nebspring 0.025
28  nebstep 20.0
29  neboutfreq 10
30  ngdsrelax 10000
31 
32  # Threshold paramters when using LAMPPS/REAXFF:
33  cithresh 1d-2
34  nebconv 8d-3
35  vsthresh 5d-2
36  # The use of these forces are recommended by the original AUTONEB article...
37  projforcetype 3
38 
39  # Reconnect linearly interpolated structures, then idppguess then improves that interpolation
40  reconnect .true.
41  idppguess .true.
42 
43  # Strip inactive molecules
44  stripinactive .true.
45 
46  # For most problems, optimise before starting:
47  optendsbefore .true.
48  optendsduring .false.
49 
50  # No restraining based on connectivity matrices at end-points.
51  nebrestrend .false.
52 
53  shimmybeads .true.
54  fraginterpol .true.
55  reactivegather .true.
56 
57  # constraints
58  dofconstraints 0
59 
60  # The platinum atom is frozen
61  atomconstraints 1
62  1
63 
64  # Use lammps, using lmp.min for minimizations and lmp.head for gradients and energies
65  pestype lammps
66  pesfile lmp.head
67  pesopttype lammps
68  pesoptfile lmp.min
69 
70  # Name of the executables
71  pesexecutable '/Users/scott/code/lammps-22Aug18/src/lmp_serial'
72  pesoptexecutable '/Users/scott/code/lammps-22Aug18/src/lmp_serial'
73 
74 As shown above, this calculation is set up to first perform a NEB calculation with 7 images. After that, the minimum energy path is further refined by addition of 23 images according to the AutoNEB algorithm.
75 
76 ##Output files
77 
78 Note that, when running AutoNEB calculations, several additional output files are produced, including:
79 
80 - *.energy-autoneb, which shows the energy of the reaction profile after each additional AutoNEB image has been added and refined (Here, the * is simply the name of the original input file).
81 
82 - *.energy-autoneb-end, which gives the energy of the final AutoNEB-refined minimum-energy path.
83 
84 - *_autonebfinal.xyz, which is the xyz structure file for the final refined reaction path after completion of NEB and AutoNEB.
