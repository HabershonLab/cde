1 ### I/O structure formats {#formats}
2 
3 Some important things to note about structure formats used in CDE:
4 
5 - Throughout CDE, we generally use simple *.xyz files as input/output formats.
6 
7 - You can use *openbabel* (http://openbabel.org/wiki/Main_Page) to convert from a wide range of files to/from xyz files.
8 
9 - A convenient program for creating initial structures in xyz format is *Avogadro* (https://avogadro.cc).
10 
11 - The coordinates in an XYZ format file in units of Angstroms.
12 
13 - If performing NEB calculations, the number of snapshots (or configurations) in the xyz file for the full path should be the same as that defined in *nimage* in the input file.
14 
15 - Each xyz frame or file looks like this:
16 
17  2
18 
19  C 0.000 0.000 0.0000
20  O 1.50 1.670 5.6708
21 
22 
23 - The first line contains the number of atoms, the second line is blank (or a comment line) and then there is a list of atoms with their atomic labels and (x,y,z) coordinates in Angstroms.
24 
25 - Different frames in an xyz file follow each other immediately, like this:
26 
27  2
28 
29  C 0.000 0.000 0.0000
30  O 1.50 1.670 5.6708
31  2
32 
33  C 0.1500 0.2166 0.007
34  O 1.56 1.679 5.7098
